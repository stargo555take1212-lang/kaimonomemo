# 買い物メモ

家族で共有する「買い物・日用品の在庫切れメモ」アプリです。
なくなったもの(卵、豚肉、薬など)を記録しておき、外出先でスマホから確認 → 買ったらチェックしてリストから消す、という使い方を想定しています。

- 「よく使うもの」を事前登録しておき、ワンタップで追加できます(卵・牛乳・常備薬など)
- スーパー・薬局など「買う場所」ごとにカテゴリ分け(自由に追加・編集可能)
- アイテムのドラッグ(ドラッグハンドル ⠿⠿ を押しながら移動)でカテゴリ間の振り分けが可能。タップでもカテゴリ選択で移動できます
- データは **Googleスプレッドシート** に保存され、家族の端末間で自動同期(20秒ごとにポーリング)
- HTML1ファイルで動作し、スマホのホーム画面にアイコン付きで追加可能(オリジナルアイコン同梱)

同梱ファイル:
- `index.html` … アプリ本体(このファイルを家族の端末に配布します)
- `Code.gs` … データ保存用の Google Apps Script(スプレッドシート側に設置します)
- `icon-512.png` / `icon-180.png` … アプリアイコン(参考用。index.html には既に埋め込み済みなので通常は使いません)

## 1. バックエンド(スプレッドシート)のセットアップ ※最初に1回だけ

1. Google スプレッドシートを新規作成する(https://sheets.google.com)
2. メニュー「拡張機能」→「Apps Script」を開く
3. 開いたエディタの `コード.gs` の中身を全部削除し、同梱の **`Code.gs`** の内容を丸ごと貼り付けて保存(Ctrl/Cmd+S)
4. 画面上部の関数選択で `setup` を選び、▶ 実行ボタンを押す
   - 初回は権限承認の画面が出るので「許可」する(自分のスプレッドシートなので安全です)
   - 実行するとシートに `Categories`・`Items` の2枚のシートができ、初期カテゴリ(スーパー・薬局)が入ります
5. 右上「デプロイ」→「新しいデプロイ」
   - 種類の選択で歯車アイコン →「ウェブアプリ」を選択
   - 実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**
   - 「デプロイ」をクリックし、表示される **ウェブアプリ URL(.../exec で終わるもの)** をコピーしておく

これでバックエンドの準備は完了です。このURLが家族全員のアプリをつなぐ「合言葉」になります。

> コードを直しても、この URL は同じままでは反映されません。`Code.gs` を編集したら「デプロイ」→「デプロイを管理」→ 該当デプロイの編集(鉛筆アイコン)→ バージョン「新バージョン」を選んで再デプロイしてください。

**すでに運用中で、後から「よく使うもの」機能を追加する場合:**
`setup()` を再実行するとカテゴリ・アイテムが消えてしまうため、代わりに関数選択で `migrateAddTemplates` を選んで実行してください(Templatesシートのみ追加されます)。実行後、上記と同様に「新バージョン」として再デプロイしてください。

## 2. アプリ(index.html)を使う

1. `index.html` を家族それぞれのスマホに配布する(LINEやメール、AirDropなどで送るのが簡単です)
2. スマホのブラウザ(Safari / Chrome)でそのファイルを開く
3. 右上の ⚙️ 設定 をタップし、手順1でコピーした URL を貼り付けて「保存」
   - 「接続テスト」で成功メッセージが出れば OK です
4. ホーム画面にアイコンを追加する
   - **iPhone (Safari)**: 共有ボタン → 「ホーム画面に追加」
   - **Android (Chrome)**: メニュー(⋮) → 「ホーム画面に追加」/「アプリをインストール」
   - オリジナルアイコンが表示されます

家族全員が同じ URL を設定すれば、誰かが追加・チェックした内容が他の端末にも(最大20秒程度で)反映されます。

### 使い方

- 画面下の入力欄に品名を入力し、上のチップで「どのお店で買うか」を選んで ＋ で追加
- なくなったものを買ったら、丸いチェックボタンをタップ → リストから消えます
- カテゴリ(お店)を移動したいときは、ドラッグハンドル `⠿⠿` を押しながらドラッグ、またはアイテム名をタップして移動先を選択
- チップの「＋ お店を追加」で新しいお店・カテゴリを追加(名前と色を選べます)
- 既存のカテゴリを長押し、または各カテゴリの ✎ アイコンで編集・削除

### 「よく使うもの」機能

- 入力欄の左にある ☆ ボタンをタップすると、入力中の品名とカテゴリで「よく使うもの」に登録できます
- リスト上部の「⭐ よく使うもの」に並ぶチップをタップするだけで、そのままリストに追加されます(入力不要)
- 「⭐ よく使うもの」の右端「✎ 管理」から、登録済みの一覧確認・削除・新規登録ができます
- 卵・牛乳・常備薬などをあらかじめ登録しておくと便利です

## 3. 今後の改修について(Git / VSCode / Claude Code)

### VSCodeに移行する手順

1. ダウンロードした `index.html` `Code.gs` `README.md` `icon-*.png` を作業用フォルダに集める
2. Gitリポジトリ化
   ```bash
   git init
   git add .
   git commit -m "初回コミット: 買い物メモ v1"
   ```
3. (任意) GitHubにpush
   ```bash
   gh repo create kaimono-memo --private --source=. --push
   ```
4. VS Codeで開く
   ```bash
   code .
   ```
   「Live Server」拡張を入れておくと、`index.html` を `http://localhost` 経由で開けて動作確認が安定します(`file://` 直開きは環境によりApps Scriptへの通信がブロックされることがあります)。
5. (任意) Apps Script側も `clasp`(Google公式CLI)でリポジトリ管理できます
   ```bash
   npm install -g @google/clasp
   clasp login
   clasp clone <スクリプトID>   # 既存プロジェクトに紐付ける場合
   ```
6. VS Code拡張の Claude Code を入れておくと、フォルダを開いた状態のままチャットで改修を指示できます

このリポジトリには `index.html`(フロントエンド)と `Code.gs`(バックエンド)の2つのソースがあります。

- `index.html` はブラウザで直接開くだけで動作確認できます(ローカルサーバー不要)。ただし `file://` で開く場合、一部ブラウザで Google Apps Script への fetch がブロックされることがあるので、その場合は VSCode の Live Server 拡張などで `http://localhost` 経由にすると安定します。
- `Code.gs` は Apps Script エディタ上で直接編集するか、[clasp](https://github.com/google/clasp)(Google公式CLI)を使うとローカルで編集して `clasp push` でデプロイでき、Git 管理もしやすくなります。
- 配色・フォントなどのデザイントークンは `index.html` 内の `<style>` 冒頭の `:root { --paper: ...; --accent: ...; }` にまとまっています。

拡張アイデア例:
- 買った履歴の一覧表示(元に戻す機能)
- 定番品を1タップで再登録するテンプレート機能
- カテゴリの並び替え(ドラッグで順番変更)
- プッシュ通知(要: Firebase等の追加バックエンド)
